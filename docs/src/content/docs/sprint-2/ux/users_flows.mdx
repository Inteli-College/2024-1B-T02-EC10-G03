# Wireframe - Fluxo dos Usuários

## Fluxo do Usuário no Sistema para Bacharel em Farmácia

1. **Login/Autenticação**
   - O bacharel em farmácia acessa o sistema utilizando suas credenciais de login.

2. **Dashboard de Dados Gerais**
   - Após o login, ele é direcionado para o dashboard principal do sistema, onde pode visualizar dados gerais sobre o funcionamento da farmácia.

3. **Estatísticas sobre as Inconsistências**
   - Ele navega para a seção de estatísticas, onde pode analisar as inconsistências identificadas no estoque de medicamentos.

4. **Visualização das Urgências**
   - Na página de estatísticas, o bacharel em farmácia tem acesso a uma seção específica que mostra as urgências atuais relacionadas aos medicamentos.

5. **Acompanhamento Geral**
   - Ele pode acompanhar o andamento das urgências e verificar se estão sendo resolvidas de forma adequada.

6. **Visualização de Inconsistências**
   - Por fim, ele pode visualizar em detalhes as inconsistências identificadas no estoque e tomar as medidas necessárias para corrigi-las.

## Fluxo do Usuário no Sistema para Técnico em Farmácia

1. **Login/Autenticação**
   - O técnico em farmácia faz login no sistema utilizando suas credenciais.

2. **Lista de Ocorrências por Ordem de Prioridade**
   - Após o login, ele é direcionado para a lista de ocorrências, onde pode visualizar as demandas de medicamentos por ordem de prioridade.

3. **Feedback de Retorno da Ocorrência**
   - Ele seleciona uma ocorrência específica e fornece feedback sobre o status e a completude do pedido de medicamentos.

4. **Visualização do Sistema - Dados e Dash**
   - Em seguida, ele acessa outras seções do sistema para visualizar dados relevantes e históricos sobre o fluxo de medicamentos.

---

## Fluxo do Usuário no Sistema para Técnico em Enfermagem

1. **Login/Autenticação**
   - O técnico em enfermagem acessa o sistema utilizando suas credenciais de login.

2. **Scannear o Pyxis**
   - Após o login, ele escaneia o Pyxis para acessar informações sobre os medicamentos disponíveis.

3. **Preencher Ocorrência**
   - Ele preenche as ocorrências relacionadas às demandas de medicamentos, indicando os itens necessários e a urgência do pedido.

4. **Visualizar Histórico**
   - Em seguida, ele pode visualizar o histórico de ocorrências e acompanhar o status dos pedidos anteriores.

5. **Acompanhar Pedidos**
   - Por fim, ele pode acompanhar o andamento dos pedidos de medicamentos e garantir que sejam atendidos de forma adequada.
